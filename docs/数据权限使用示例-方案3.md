# 数据权限使用示例 - 方案3（Service层注解）

## 📋 实现说明

本次实施采用**方案3：在Service层标注注解**，核心思路是：
- 将 `@DataPermission` 注解从 Mapper 层移至 Service 层
- Spring AOP 可以拦截 Spring Bean（Service）的方法
- 在 Service 方法执行前，AOP 将注解存入 ThreadLocal
- MyBatis-Flex 的 Dialect 在生成 SQL 时从 ThreadLocal 获取注解并修改 SQL

## ✅ 核心组件

### 1. DataPermissionAutoConfiguration（自动配置类）
**位置**: `mdp-base/md-db-mybatis-flex/src/main/java/top/mddata/base/mybatisflex/config/DataPermissionAutoConfiguration.java`

**作用**: 
- 注册 `DataPermissionAspect` 为 Spring Bean
- 启用 AspectJ 自动代理（`@EnableAspectJAutoProxy`）
- 通过 Spring Boot 自动配置机制自动加载

### 2. DataPermissionAspect（切面类）
**位置**: `mdp-base/md-db-mybatis-flex/src/main/java/top/mddata/base/mybatisflex/datapermission/DataPermissionAspect.java`

**作用**: 
- 拦截带有 `@DataPermission` 注解的方法
- 在方法执行前将注解存入 ThreadLocal
- 在方法执行后清理 ThreadLocal

## 🚀 使用示例

### 示例1：在 Controller 中使用

```java
@RestController
@RequestMapping("/config")
@RequiredArgsConstructor
public class ConfigController {
    
    private final ConfigService configService;
    
    /**
     * 带数据权限的配置列表查询
     */
    @GetMapping("/listWithPermission")
    public R<List<Config>> listWithPermission() {
        // 构建查询条件
        QueryWrapper queryWrapper = QueryWrapper.create()
                .eq(Config::getState, true)
                .orderBy(Config::getCreatedAt, false);
        
        // 调用带数据权限的方法
        // 会根据当前用户的数据权限自动过滤数据
        List<Config> list = configService.listWithPermission(queryWrapper);
        
        return R.success(list);
    }
}
```

### 示例3：自定义数据权限配置

```java
@Service
public class CustomServiceImpl extends SuperServiceImpl<CustomMapper, CustomEntity> {
    
    /**
     * 自定义数据权限配置
     */
    @Override
    @DataPermission(
        tableAlias = "t",           // 表别名
        deptId = "dept_id",         // 部门ID字段
        userId = "create_user",     // 用户ID字段
        roleId = "role_id",         // 角色ID字段
        deptTableAlias = "sys_dept",           // 部门表别名
        roleDeptTableAlias = "sys_role_dept"   // 角色部门关联表别名
    )
    @Transactional(readOnly = true)
    public List<CustomEntity> listWithPermission(QueryWrapper queryWrapper) {
        return getMapper().selectListByQuery(queryWrapper);
    }
}
```

## 🔄 执行流程

```
1. Controller/Service 调用 configService.listWithPermission(queryWrapper)
   ↓
2. Spring AOP 拦截到 @DataPermission 注解
   ↓
3. DataPermissionAspect.beforeMethod() 执行
   THREAD_LOCAL.set(dataPermission)  ✅ 注解存入 ThreadLocal
   ↓
4. 进入 listWithPermission 方法体
   ↓
5. 调用 getMapper().selectListByQuery(queryWrapper)
   ↓
6. MyBatis-Flex 调用 DataPermissionDialect.forSelectByQuery()
   ↓
7. DataPermissionDialect 从 ThreadLocal 获取注解
   DataPermission dataPermission = DataPermissionAspect.currentDataPermission();
   ↓
8. 根据用户角色和数据权限范围，自动修改 SQL：
   - ALL: 不添加任何条件
   - DEPT: WHERE dept_id = xxx
   - DEPT_AND_CHILD: WHERE dept_id IN (SELECT id FROM sys_dept WHERE ...)
   - SELF: WHERE create_user = xxx
   - CUSTOM: WHERE dept_id IN (SELECT dept_id FROM sys_role_dept WHERE role_id = xxx)
   ↓
9. 执行修改后的 SQL，返回过滤后的数据
   ↓
10. 方法执行完毕
    ↓
11. DataPermissionAspect.afterMethod() 执行
    THREAD_LOCAL.remove()  ✅ 清理 ThreadLocal
```

## ⚠️ 注意事项

### 1. 必须调用新方法
```java
// ❌ 错误：直接调用 Mapper 方法，不会有数据权限
List<Config> list = configService.getMapper().selectListByQuery(queryWrapper);

// ✅ 正确：调用 Service 的 listWithPermission 方法
List<Config> list = configService.listWithPermission(queryWrapper);
```

### 2. 继承的 list 方法不会有数据权限
```java
// ❌ 这些方法继承自基类，没有 @DataPermission 注解
List<Config> list1 = configService.list(queryWrapper);
List<Config> list2 = configService.listByIds(ids);

// ✅ 必须使用新方法
List<Config> list3 = configService.listWithPermission(queryWrapper);
```

### 3. 数据权限依赖用户上下文
```java
// 确保当前用户已登录，并且有角色和数据权限配置
// DataPermissionFilter.getCurrentUser() 需要返回有效的用户信息
```

## 📊 数据权限范围说明

| 权限范围 | 说明 | SQL 示例 |
|---------|------|---------|
| ALL | 全部数据 | 不添加任何条件 |
| DEPT | 本部门数据 | `WHERE dept_id = #{currentDeptId}` |
| DEPT_AND_CHILD | 本部门及以下 | `WHERE dept_id IN (SELECT id FROM sys_dept WHERE id = xxx OR find_in_set(xxx, ancestors))` |
| SELF | 仅本人数据 | `WHERE create_user = #{currentUserId}` |
| CUSTOM | 自定义数据 | `WHERE dept_id IN (SELECT dept_id FROM sys_role_dept WHERE role_id = #{roleId})` |

## 🔧 如何为其他实体添加数据权限

### 步骤1：在 Service 接口中添加方法
```java
public interface UserService extends SuperService<User> {
    // 其他方法...
    
    /**
     * 带数据权限的查询用户列表
     */
    List<User> listWithPermission(QueryWrapper queryWrapper);
}
```

### 步骤2：在 Service 实现类中实现方法
```java
@Service
public class UserServiceImpl extends SuperServiceImpl<UserMapper, User> implements UserService {
    
    @Override
    @DataPermission  // 添加数据权限注解
    @Transactional(readOnly = true)
    public List<User> listWithPermission(QueryWrapper queryWrapper) {
        return getMapper().selectListByQuery(queryWrapper);
    }
}
```

### 步骤3：使用新方法
```java
// Controller 或其他 Service 中调用
List<User> users = userService.listWithPermission(queryWrapper);
```

## 🎯 总结

- ✅ **优点**：实现简单，符合 Spring AOP 设计理念
- ⚠️ **注意**：必须调用新的 `listWithPermission` 方法才能应用数据权限
- 📝 **建议**：对于需要数据权限的查询，统一使用 `listWithPermission` 方法

## 🔧 问题修复记录

### 问题描述
调用 `listWithPermission` 方法时，无法进入 `DataPermissionAspect` 的任何方法。

### 根本原因
`DataPermissionAspect` 类只有 `@Aspect` 注解，但**没有被注册为 Spring Bean**，导致 Spring AOP 无法激活该切面。

### 解决方案
1. 创建 `DataPermissionAutoConfiguration` 自动配置类
2. 在配置类中注册 `DataPermissionAspect` Bean
3. 添加 `@EnableAspectJAutoProxy` 启用 AspectJ 自动代理
4. 将配置类添加到 Spring Boot 自动配置列表
